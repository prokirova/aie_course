## 2. Baseline-модели

Были реализованы два baseline-подхода:

1. **DummyClassifier (strategy='most_frequent')**  
   - Accuracy: **0.XX**
   - F1: **0.XX**  
   *Интерпретация*: модель всегда предсказывает самый частый класс. Служит «нулевой гипотезой» — любая осмысленная модель должна её превосходить.

2. **LogisticRegression (в Pipeline со StandardScaler)**  
   - Accuracy: **0.XX**
   - F1: **0.XX**
   - ROC-AUC: **0.XX** (только для бинарной задачи)  
   *Интерпретация*: линейная модель уже даёт улучшение относительно Dummy, что говорит о наличии линейно разделимых закономерностей в данных.

Обе baseline-модели обучались **без подбора гиперпараметров**, только на `X_train`/`y_train`, оценка — на `X_test`.

## 3. Основные модели

Реализованы и протестированы следующие модели:

- **DecisionTreeClassifier** с контролем сложности через `max_depth` и `min_samples_leaf`.
- **RandomForestClassifier** с настройкой `n_estimators`, `max_depth`.
- **GradientBoostingClassifier** (или AdaBoost/HistGradientBoosting) с подбором `learning_rate`, `n_estimators`.
- **(Опционально) StackingClassifier** на основе трёх базовых моделей и логистической регрессии в качестве мета-классификатора.

Для каждой модели (кроме Dummy) проводился **подбор гиперпараметров на обучающей выборке** с использованием `GridSearchCV` и кросс-валидации (`cv=5`). Использовалась метрика **f1 (binary)** / **f1_macro (multiclass)**, соответствующая целевой метрике качества.

Финальная оценка всех моделей выполнена **единожды на тестовой выборке**.

4. `feature_D` — важность ≈ 0.XX

**Интерпретация**:  
Признак `feature_A` ожидаемо оказался наиболее значимым — он напрямую связан с целевой переменной по смыслу (например, «уровень дохода» для предсказания дефолта).  
Остальные важные признаки также согласуются с предметной областью. Отсутствие в топе, например, `feature_Z` говорит о его слабой предсказательной силе — возможно, его можно исключить для упрощения модели.
