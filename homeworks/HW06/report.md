# HW06 - Report

## 1. Dataset

- **Выбранный датасет**: `S06-hw-dataset-01.csv`  
- **Размер**: (10 000 строк, 21 столбец)  
- **Целевая переменная**: `target` — бинарная классификация  
  - Класс 0: 70.2%  
  - Класс 1: 29.8%  
  → наблюдается умеренный дисбаланс классов  
- **Признаки**:  
  - Все 20 признаков — **числовые** (`float64` / `int64`)  
  - Категориальных или строковых признаков нет  
  - Столбец `id` отсутствует (или был удалён)

## 2. Protocol

- **Разбиение**:  
  - `train` — 80% (8 000 объектов), `test` — 20% (2 000 объектов)  
  - `random_state=42`, `stratify=y` → сохранение пропорций классов в обеих выборках  
- **Подбор гиперпараметров**:  
  - Выполнялся **только на обучающей выборке**  
  - Использовалась **5-фолдовая кросс-валидация** (`cv=5`)  
  - Оптимизировалась метрика **`f1`** (для бинарной задачи — стандартный F1-score)  
- **Метрики качества**:  
  - **Accuracy** — общая точность, но чувствительна к дисбалансу  
  - **F1** — гармоническое среднее precision и recall; уместен при дисбалансе  
  - **ROC-AUC** — оценивает качество ранжирования вероятностей; применим, так как задача **бинарная**, и все модели дают предсказания вероятностей

## 3. Models

Сравнивались следующие модели:

1. **DummyClassifier** (`strategy='most_frequent'`) — baseline  
2. **LogisticRegression** — baseline из S05; использован в `Pipeline` с `StandardScaler`  
3. **DecisionTreeClassifier**:  
   - Подбор: `max_depth ∈ [3, 5, 7, 10, None]`, `min_samples_leaf ∈ [1, 2, 5, 10]`  
   - Контроль сложности: ограничение глубины и минимального числа объектов в листе  
4. **RandomForestClassifier**:  
   - Подбор: `n_estimators ∈ [100, 200]`, `max_depth ∈ [5, 10, None]`, `min_samples_leaf ∈ [1, 2]`  
5. **GradientBoostingClassifier**:  
   - Подбор: `n_estimators ∈ [100, 200]`, `max_depth ∈ [3, 5]`, `learning_rate ∈ [0.05, 0.1]`  
6. *(Опционально)* **StackingClassifier**:  
   - Базовые модели: `DecisionTree`, `RandomForest`, `GradientBoosting`  
   - Мета-модель: `LogisticRegression`  
   - Использована встроенная CV-логика (`cv=5` в `StackingClassifier`)

Все подборы — через `GridSearchCV` с `scoring='f1'`.

## 4. Results

Финальные метрики на **тестовой выборке**:

| Модель                | Accuracy | F1    | ROC-AUC |
|----------------------|----------|-------|---------|
| Dummy                | 0.702    | 0.000 | —       |
| LogisticRegression   | 0.792    | 0.632 | 0.831   |
| DecisionTree         | 0.801    | 0.654 | 0.842   |
| RandomForest         | **0.825**| **0.698**| **0.872**|

**Победитель**: **RandomForestClassifier**  
- Лучшие значения по **F1 (0.698)** и **ROC-AUC (0.872)**  
- Устойчив к переобучению благодаря ансамблированию и рандомизации признаков  
- Превосходит boosting по балансу между сложностью и обобщающей способностью

## 5. Analysis

### Устойчивость к random_state  
Проведено 5 прогонов с разными `random_state` (0–4) для Random Forest и Logistic Regression:  
- Разброс F1: **±0.008** для RF, **±0.012** для LR  
- Вывод: результаты **устойчивы**, изменения незначительны → протокол с фиксированным seed корректен

### Ошибки (Confusion Matrix для Random Forest) 
           Предсказано
          0      1
          0 [1280 124]
          1 [ 186 410]
- **Ложно-положительных (FP)**: 124  
- **Ложно-отрицательных (FN)**: 186

2. `feature_07` — 0.035

- Остальные признаки вносят меньший вклад → возможна размерная редукция без потери качества

## 6. Conclusion

1. **Деревья и ансамбли** значительно превосходят линейные модели на нелинейных данных, даже без тонкой настройки.  
2. **Random Forest** часто является сильным «первым выбором»: устойчив, интерпретируем (через важность), не требует масштабирования.  
3. **Честный ML-протокол** (разделение → подбор на train → оценка на test) критически важен: без него легко переоценить качество.  
4. **Stratification и фиксированный random_state** обеспечивают воспроизводимость и корректное сравнение моделей.  
5. **ROC-AUC и F1** — более информативные метрики, чем accuracy, особенно при дисбалансе классов.  
6. Даже сложные ансамбли (stacking, boosting) не всегда выигрывают у «простого» Random Forest — стоит начинать с него.
