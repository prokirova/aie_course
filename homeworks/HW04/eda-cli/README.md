# S04 – eda_cli: мини-EDA для CSV

Небольшое CLI-приложение для базового анализа CSV-файлов.
Используется в рамках Семинара 03 курса «Инженерия ИИ».

## Требования

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) установлен в систему

## Инициализация проекта

В корне проекта (S03):

```bash
uv sync
```

Эта команда:

- создаст виртуальное окружение `.venv`;
- установит зависимости из `pyproject.toml`;
- установит сам проект `eda-cli` в окружение.

## Запуск CLI

### Краткий обзор

```bash
uv run eda-cli overview data/example.csv
```

## Доступные команды

- `overview` — выводит краткую сводку по датасету в терминал.
- `report` — генерирует подробный EDA-отчёт в Markdown и графиках.

## Новые параметры команды `report`

Добавлены следующие опции, влияющие на содержание отчёта:

- `--max-hist-columns N` — ограничивает количество числовых колонок, для которых строятся гистограммы (по умолчанию: 6).
- `--top-k-categories K` — задаёт, сколько самых частых значений отображать для каждой категориальной колонки (по умолчанию: 5).
- `--min-missing-share T` — устанавливает порог доли пропусков (в диапазоне от 0.0 до 1.0); колонки с пропусками выше этого порога выделяются в отдельный раздел «Проблемные колонки» в `report.md` (по умолчанию: 0.1, то есть 10%).

Эти параметры также отображаются в разделе **«Параметры генерации отчёта»** внутри `report.md`.

## Пример вызова

```bash
uv run eda-cli report data/sales.csv \
  --out-dir reports_sales \
  --max-hist-columns 8 \
  --top-k-categories 10 \
  --min-missing-share 0.25
