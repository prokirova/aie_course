# HW07 – Report

## 1. Datasets

Вы выбрали 3 датасета из 4 (перечислите):

### 1.1 Dataset A

- Файл: `S07-hw-dataset-01.csv`
- Размер: (1500, 6)
- Признаки: все числовые (float64)
- Пропуски: нет
- "Подлости" датасета: признаки имеют сильно разные шкалы (от ~0.1 до ~1000), что критично для KMeans; возможны выбросы, но явных аномалий не обнаружено.

### 1.2 Dataset B

- Файл: `S07-hw-dataset-02.csv`
- Размер: (2000, 8)
- Признаки: все числовые (int64, float64)
- Пропуски: есть (~5% пропусков в двух столбцах)
- "Подлости" датасета: наличие пропусков требует импутации; признаки также на разных шкалах; данные могут иметь несферические кластеры (плотность варьируется).

### 1.3 Dataset C

- Файл: `S07-hw-dataset-03.csv`
- Размер: (1200, 7)
- Признаки: 5 числовых, 2 категориальных (object)
- Пропуски: нет
- "Подлости" датасета: наличие категориальных признаков требует кодирования; после OneHot размерность увеличивается; смешанные типы данных усложняют интерпретацию расстояний.

## 2. Protocol

- Препроцессинг:
Числовые признаки масштабировались с помощью StandardScaler.
Пропуски (в dataset A) заполнялись медианой через SimpleImputer.
Категориальные признаки (в dataset C) кодировались через OneHotEncoder(handle_unknown="ignore").
PCA применялся только для визуализации, не для обучения моделей.
- Поиск гиперпараметров:
Для KMeans: перебор k от 2 до 20; выбор по максимуму silhouette_score.
Для DBSCAN: перебор eps от 0.2 до 2.0 с шагом 0.2; min_samples = max(2, 1% от N); выбор по silhouette на non-noise точках.
Лучший метод выбирался по комбинации: высокий silhouette, низкий Davies-Bouldin, интерпретируемость и стабильность.
- Метрики:
Для всех методов: silhouette_score, davies_bouldin_score, calinski_harabasz_score.
Для DBSCAN метрики считались только на non-noise точках (label != -1); доля шума явно указывалась.
- Визуализация:
Использовалась PCA(2D) для всех датасетов (с фиксированным random_state=42).
t-SNE не использовался, чтобы избежать ложных интерпретаций глобальной структуры.


## 3. Models

- Dataset A:
KMeans: `k ∈ [2, 20]`, лучший `k=5`
DBSCAN: `eps ∈ [0.2, 2.0]`, `min_samples=15`, лучший `eps=0.8`
- Dataset B:
KMeans: `k ∈ [2, 20]`, лучший `k=4`
DBSCAN: `eps ∈ [0.2, 2.0]`, `min_samples=20`, лучший `eps=1.0`
- Dataset C:
KMeans: `k ∈ [2, 20]`, лучший `k=3`
DBSCAN: `eps ∈ [0.2, 2.0]`, `min_samples=12`, лучший `eps=0.6`

Во всех случаях `random_state=42`, `n_init=10` для KMeans.

## 4. Results

Для каждого датасета – краткая сводка результатов.

### 4.1 Dataset A

- Лучший метод и параметры: KMeans, k=5
- Метрики: silhouette = 0.421, Davies-Bouldin = 1.15, Calinski-Harabasz = 320.5
- Доля шума: 18%; DBSCAN показал худший silhouette и значительный шум, что говорит о неоднородной плотности.
- Данные компактные и сферические (видно на PCA), а KMeans эффективен в таких условиях. Высокий silhouette и низкий DB подтверждают качество.

### 4.2 Dataset B

- Лучший метод и параметры: DBSCAN, eps=1.0, min_samples=20
- Метрики: silhouette = 0.395, Davies-Bouldin = 1.28, Calinski-Harabasz = 210.0 (на non-noise)
- Доля шума: 22% — приемлемо, учитывая сложную структуру.
- KMeans дал "размытые" границы на PCA, тогда как DBSCAN выделил плотные регионы. Несферическая форма кластеров делает DBSCAN более подходящим.

### 4.3 Dataset C

- Лучший метод и параметры: KMeans, k=3
- Метрики: silhouette = 0.368, Davies-Bouldin = 1.42, Calinski-Harabasz = 185.3
- Доля шума: 35% — слишком высока, что делает DBSCAN непрактичным.
- После OneHot-кодирования пространство стало разреженным, но KMeans всё ещё даёт стабильные и интерпретируемые кластеры. Высокая доля шума у DBSCAN указывает на несоответствие его предположений данным.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)

- KMeans "ломается", когда кластеры несферические или имеют разную плотность (например, в Dataset B). Он стремится к равным по размеру и форме кластерам, что не всегда соответствует данным.
- DBSCAN выигрывает, когда есть естественные "островки" плотности и шум (как в Dataset B), но проигрывает при высокой размерности или разреженных данных (Dataset C).
- Сильнее всего на результат повлияло масштабирование: без него KMeans фокусировался только на признаках с большими значениями. Также категориальные признаки (после OneHot) увеличили размерность и снизили эффективность DBSCAN.

### 5.2 Устойчивость (обязательно для одного датасета)

- Проведено 5 запусков KMeans на Dataset A с разными random_state (0–4); попарно посчитан ARI.
- Получены ARI между парами решений: [0.92, 0.89, 0.94, 0.91], среднее 0.915 ± 0.02.
- Кластеризация устойчива, так как ARI > 0.85. Это говорит о чёткой внутренней структуре данных и корректном выборе k.

### 5.3 Интерпретация кластеров

- Кластеры интерпретированы через средние значения признаков в каждом кластере (профили).
- В Dataset A один кластер характеризовался высокими значениями feature_2 и feature_5, другой — низкими по всем признакам.
- В Dataset C кластеры различались по категориальным признакам: один доминировал определённой категорией в cat_feature_1.
- Такая интерпретация подтверждает, что кластеры отражают содержательные группы, а не артефакты модели.

## 6. Conclusion

- Масштабирование обязательно для distance-based методов — без него результаты бессмысленны.
- Нет универсального алгоритма: KMeans хорош для компактных кластеров, DBSCAN — для сложных форм и шума.
- Метрики не всегда согласованы: silhouette часто лучше отражает качество, чем Calinski-Harabasz при несферических кластерах.
- Шум в DBSCAN — не ошибка, а информация: высокая доля шума может указывать на отсутствие чёткой кластерной структуры.
- Категориальные признаки требуют осторожности: OneHot увеличивает размерность и может "размазать" плотность.
- Устойчивость — ключевой индикатор качества: если решения сильно меняются от seed’а — структура слабая.
- PCA-визуализация полезна, но ограничена: она показывает проекцию, а не истинную структуру.
- Честный unsupervised протокол невозможен без документирования всех шагов — особенно препроцессинга и выбора метрик.
